Stage 3-17-4 · Zero-Phase Zero Padding 關卡 3-17-4 · 零相位補零 SASP §3.17.4 ↗

Appending zeros shifts the window's centre of energy by $N/2$. This produces a huge linear-phase slope in the FFT even when the underlying signal is symmetric. Zero-phase padding keeps the centre at index 0.

把零加在後面會讓視窗能量中心位移 $N/2$，即使訊號是對稱的，FFT 也會冒出一條巨大的線性相位斜率。零相位補零把能量中心維持在索引 0。

The layout

排法

  1. Centre the windowed data at index 0: left half at positive indices $0\ldots N/2-1$, right half at $M-N/2\ldots M-1$ (Matlab calls this layout fftshift).
  2. 把加窗資料中心置於索引 0：左半放正索引 $0\ldots N/2-1$、右半放 $M-N/2\ldots M-1$（Matlab 叫 fftshift）。
  3. Fill the middle with zeros.
  4. 中間填零。
  5. FFT as usual.
  6. 照常 FFT。
LAYOUT IN M-POINT BUFFER (N ODD)

Result: the DFT is real if the input was symmetric about $n=0$. No linear phase. This is what you want for tracking phase changes of sinusoids across time — otherwise the fixed linear phase dominates any actual motion.

結果：如果原資料對 $n=0$ 對稱，DFT 為純實數，沒有線性相位。 跨 frame 追蹤正弦波相位變化時你要的就是這個 — 否則固定的線性相位會壓過真正的相位運動。

Phase vocoder, sinusoidal modelling, group-delay estimation. Any application where you compare phase across frames needs zero-phase layout. Audio-display and magnitude-only applications don't need it.

相位聲碼器、正弦建模、群延遲估計。任何需要跨 frame 比較相位的應用都要零相位排法。只看振幅的顯示類應用則不用。
