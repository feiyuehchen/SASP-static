Zero padding is routine — a few tiny pitfalls to keep in mind.

補零很日常，但有幾個小陷阱要避開。

Four common mistakes

四個常見錯誤

  1. Pad after windowing, not before. Windowing a zero-padded signal can taper the zeros, which distorts the frame.
  2. 先加窗、再補零，不要顛倒。補零後才加窗會把零段也拉成斜坡，毀掉 frame。
  3. Match the zero sample values to the DC assumption. If the signal has a non-zero DC component, padding with zeros introduces an edge. Subtract the mean first if you care.
  4. 補零值要與 DC 假設一致。訊號有非零平均時補零會產生邊緣。在意就先扣平均。
  5. Amplitude scaling depends on convention. Some libraries scale by $1/M$, others by $1/\sqrt{M}$, others not at all. Padding does not change the scaling — but check what your tool does.
  6. 振幅縮放看函式庫慣例。有的除以 $M$、有的除以 $\sqrt{M}$、有的不除。補零不改變縮放，但先查清楚你用的工具。
  7. Padding hides leakage, not eliminates it. The leakage shape is set by your window, not by the FFT length.
  8. 補零只是把洩漏畫得清楚，不會消除洩漏。洩漏形狀由視窗決定，與 FFT 長度無關。

What zero-padding does and doesn't do

補零做了什麼、沒做什麼

DOES: Sample the DTFT more densely; make plots smooth; enable peak interpolation; prevent circular-convolution artefacts in FFT-based filtering.
DOES NOT: Increase frequency resolution (that's window length); reduce spectral leakage (that's window shape); add information (just interpolation). 做了：更密地取樣 DTFT；讓圖變平滑；支援峰值內插；在 FFT 濾波時避免循環卷積瑕疵。
沒做：提升頻率解析度（那是視窗長度）；降低頻譜洩漏（那是視窗形狀）；加進任何新資訊（只是內插）。

Zero padding = sampling the DTFT more densely. It changes the display of the spectrum without changing the underlying information.

補零 = 更密地取樣 DTFT。改變的是頻譜的顯示，不改變底層的資訊。
