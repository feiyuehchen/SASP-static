For a plot you can read with the eye, zero-pad so the FFT is 4× to 8× the window length.

要畫出眼睛看得清楚的頻譜圖，補零讓 FFT 長度是視窗的 4× 到 8×。

Without padding, the FFT samples are spaced at $2\pi/N$ — roughly the Hann window's main-lobe half-width. You only get two samples across a lobe, so every peak looks like a jagged zigzag. Padding to $M=4N$ or $8N$ lays down 4 or 8 samples inside the same lobe and the curve becomes smooth enough to read.

不補零時，FFT 樣本間距是 $2\pi/N$ — 大約是 Hann 視窗主瓣半寬。 一個主瓣只有兩個樣本，每個峰都像鋸齒。 補零到 $M=4N$ 或 $8N$，同一個主瓣內有 4 或 8 個樣本，曲線就能一眼讀懂。

FFT cost scales as $M\log M$. Going from $N$ to $8N$ multiplies the work by about 11×, which is usually fine for plots but might hurt real-time applications. Use the minimum $M$ you need for the decision you are making (display, peak picking, etc.).

FFT 計算量是 $M\log M$。從 $N$ 變 $8N$ 大約變 11 倍，畫圖通常沒差；即時應用要注意。 依使用目的（顯示、找波峰…）挑剛好夠用的 $M$。
