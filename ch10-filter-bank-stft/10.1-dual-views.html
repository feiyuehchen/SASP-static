Stage 10-1 · Dual Views of the STFT 關卡 10-1 · STFT 雙重觀點 SASP §10.1 ↗

The STFT is a 2-D array $X_m[k]$ indexed by time-frame $m$ and frequency-bin $k$. This stage shows that you can read that array two equivalent ways — row by row (OLA) or column by column (filter bank) — and that the reading you pick dictates how cheap or expensive each modification is.

STFT 是一個以時間 frame $m$ 與頻率 bin $k$ 為索引的二維陣列 $X_m[k]$。本關展示兩種等價讀法 — 一列一列讀（OLA）或一行一行讀（濾波器組）— 而你選哪種讀法，直接決定每種修改操作是便宜還是昂貴。

Two equivalent readings of the same matrix

同一個矩陣的兩種等價讀法

Write the analysis formula once:

分析公式只寫一次：

STFT (ANALYSIS)
$$X_m[k]\;=\;\sum_{n=-\infty}^{\infty} x[n]\;w[n-mR]\;e^{-j 2\pi k n / N}$$

where $w[\cdot]$ is the analysis window, $R$ is the hop, and $N$ is the FFT length. Fix $m$ and vary $k$ — you get one frame's spectrum. Fix $k$ and vary $m$ — you get one bin's time history.

其中 $w[\cdot]$ 是分析視窗， $R$ 是跳距， $N$ 是 FFT 長度。固定 $m$、讓 $k$ 跑 — 拿到某個 frame 的頻譜；固定 $k$、讓 $m$ 跑 — 拿到某個 bin 的時間歷程。

Fix $m$, read across $k$. Row $m$ is a single short-time spectrum — the DFT of the windowed frame $x[n]w[n-mR]$. Reconstruction is "IFFT each row, then overlap-add".

固定 $m$，沿 $k$ 讀。第 $m$ 列是一個短時頻譜 — 加窗 frame $x[n]w[n-mR]$ 的 DFT。重建方式：「每列 IFFT、再重疊相加」。

Fix $k$, read across $m$. Column $k$ is the output of a bandpass filter centred at $\omega_k=2\pi k/N$, whose impulse response is a modulated copy of the window. Reconstruction is "sum all channels (with optional synthesis filters)".

固定 $k$，沿 $m$ 讀。第 $k$ 行是一個中心頻率在 $\omega_k=2\pi k/N$ 的帶通濾波器輸出，其脈衝響應為視窗的調變拷貝。重建方式：「所有通道相加（可選配合成濾波器）」。

Why carry both interpretations?

為什麼要同時有兩種解讀？

Certain modifications are trivial in one view and awkward in the other:

某些修改在一種觀點下很自然，在另一種觀點下會很彆扭：

WHEN TO USE WHICH
OLA-natural: frame-based edits (per-frame spectral whitening, noise gates, phase-reset TSM),
signal lengths that must change (time-scale modification).

FB-natural: per-band edits (equalisation, constant-Q reshaping, per-channel AGC),
analysis → filtering loops (source separation, gammatone pipelines),
and anything where you want to reason about each bin as a bandpass filter. OLA 自然：以 frame 為單位的修改（逐 frame 頻譜白化、噪聲閘、相位重置的 TSM）、
以及會改變訊號長度的操作（時間伸縮）。

FB 自然：各頻帶獨立的修改（等化、constant-Q 重塑、逐通道 AGC）、
「分析 → 濾波」迴路（聲源分離、gammatone 流水線），
以及任何「把每個 bin 當成帶通濾波器來推理」的情境。

Same data, different lens. The OLA and FB views compute exactly the same $X_m[k]$ — there is no approximation. Switching lens only changes which conditions for perfect reconstruction are easy to state: OLA demands COLA on the window; FB demands Nyquist(N) on the window. They are Poisson-summation duals of each other — we'll prove it in §10.6.

同一筆資料、兩種濾鏡。OLA 與 FB 兩種觀點算出的 $X_m[k]$ 完全相同 — 沒有任何近似。切換觀點只會改變「完美重建條件」看起來怎麼寫：OLA 要求視窗滿足 COLA；FB 則要求視窗滿足 Nyquist(N) 條件。兩者互為 Poisson 求和對偶 — §10.6 會證。
