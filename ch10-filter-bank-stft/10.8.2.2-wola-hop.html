Stage 10-8-2-2 · Hop Sizes for WOLA 關卡 10-8-2-2 · WOLA 的跳距選擇 SASP §10.8.2.2 ↗

WOLA (weighted overlap-add) uses both an analysis and a synthesis window. The PR condition (§10.8.1) is now on the product $w \cdot w_s$. Work out hop sizes $R$ for common analysis/synthesis pairs — this is what lets modern codecs use $\sqrt{\text{Hann}}$ + $\sqrt{\text{Hann}}$ = Hann at $R=N/2$.

WOLA（加權重疊相加）同時用分析與合成視窗。§10.8.1 的 PR 條件現在作用在乘積 $w \cdot w_s$。本節推導常見分析/合成視窗對的跳距 $R$ — 這就是現代編解碼採用 $\sqrt{\text{Hann}}$ + $\sqrt{\text{Hann}}$ = Hann 在 $R=N/2$ 的理由。

WOLA PR condition, restated

重申 WOLA PR 條件

PRODUCT-COLA
$$\sum_m w[n - mR]\,w_s[n - mR]\;=\;A\quad\text{(const in }n\text{)}$$

$w$ 與 $w_s$ 形狀可不同。PR 只要求兩者乘積在跳距 $R$ 下為 COLA。常見組合：

Four practical WOLA patterns

四種實用 WOLA 配法

1. OLA: $w_s = 1$. Product = $w$. Need COLA-on-$w$. Pure analysis windowing — synthesis is free.
2. Same window: $w_s = w$. Product = $w^2$. Need COLA-on-$w^2$. For Hann, $w^2$ has the same period as $w$ ⇒ $R=M/2$ works.
3. Square-root: $w = w_s = \sqrt{v}$ for some target COLA window $v$. Product = $v$ which is COLA by design. MP3/AAC/Vorbis use $\sqrt{\sin}$ windows this way.
4. Asymmetric (dual): $w_s[n] = c/\sum_m w[n-mR]$ — the canonical "dual" window. Always gives PR regardless of $w$. Useful when $w$ is fixed (e.g. by analysis requirements) and synthesis must follow. 1. OLA：$w_s = 1$。乘積 $= w$。需 $w$ 滿足 COLA。純分析加窗，合成不受限。
2. 同視窗：$w_s = w$。乘積 $= w^2$。需 $w^2$ 滿足 COLA。以 Hann 為例，$w^2$ 與 $w$ 同週期 ⇒ $R=M/2$ 可用。
3. 開根號：$w = w_s = \sqrt{v}$，$v$ 為目標 COLA 視窗。乘積 $= v$ 天然 COLA。MP3/AAC/Vorbis 以 $\sqrt{\sin}$ 視窗實作。
4. 非對稱（對偶）：$w_s[n] = c/\sum_m w[n-mR]$ — 標準「對偶」視窗。任意 $w$ 都能 PR。適合 $w$ 已被分析端釘死、合成只能配合的場合。

Latency and real-time implications

延遲與即時應用的影響

In real-time systems, the WOLA pipeline adds $M - R$ samples of output buffer delay (you need the next $M-R$ samples of overlap before you can release the first frame's contribution). Smaller hop $R$ → more overlap → more latency. Choose the largest $R$ that still anti-aliases within your noise budget.

即時系統中，WOLA 流水線增加 $M - R$ 樣本的輸出緩衝延遲（必須等到下 $M-R$ 個重疊樣本到齊才能釋放首 frame 的貢獻）。$R$ 越小 → 重疊越多 → 延遲越多。選能在噪音預算內抗混疊的最大 $R$。

Product-COLA gives you two degrees of freedom. Analysis window controls what you see; synthesis window controls what you render. Their product controls reconstruction. Design around this triangle.

Product-COLA 提供兩個自由度。分析視窗決定「你看到什麼」；合成視窗決定「你輸出什麼」。兩者乘積決定「能否還原」。圍繞這個三角形做設計。
