Stage 10-2-1 · Computational Examples in Matlab 關卡 10-2-1 · Matlab 計算範例 SASP §10.2.1 ↗

Show three implementations in Matlab/Octave — direct FIR filter bank, FFT-based STFT, and "polyphase lite" — that compute the same numbers. Compare their runtimes and round-trip errors, so you can justify the FFT shortcut with hard numbers instead of hand-waving.

用 Matlab/Octave 展示三種實作 — 直接 FIR 濾波器組、FFT-based STFT、以及 "polyphase 輕量版" — 它們計算出完全相同的數字。比對執行時間與來回誤差，讓 FFT 捷徑有具體數據背書而非嘴砲。

Method A · Direct $N$-channel FIR (the brute-force FB)

方法 A · 直接 $N$ 通道 FIR（暴力 FB）

% Direct STFT filter bank — O(N^2) per output frame
% x: input length L;  w: length-N window;  R: hop;  N: # of bands
function Xk = stft_fb_direct(x, w, R, N)
  L  = length(x);
  M  = floor((L - N) / R) + 1;     % # of frames
  Xk = zeros(N, M);
  for k = 0 : N-1
    hk = w(:) .* exp( 1j * 2*pi*k*(0:N-1)' / N );   % channel-k impulse resp.
    for m = 0 : M-1
      seg = x(m*R + (1:N));                         % current frame
      Xk(k+1, m+1) = hk.' * seg(:);                 % inner product = conv at mR
    end
  end
end

Nested loop: $N\cdot M \cdot N = N^2 M$ complex multiplies. For 10 s at 44.1 kHz, $N=1024$, $R=256$: $\approx 1.7\cdot 10^9$ ops. Slow, but it makes the FB structure explicit — every $k$ has its own FIR.

三層迴圈：$N\cdot M \cdot N = N^2 M$ 次複乘。10 秒、44.1 kHz、$N=1024$、$R=256$ 時約 $1.7\cdot 10^9$ 次。慢，但 FB 結構看得很清楚 — 每個 $k$ 有自己的 FIR。

Method B · One FFT per frame (the fast STFT)

方法 B · 每 frame 一次 FFT（fast STFT）

% STFT via FFT — O(N log N) per output frame
function Xk = stft_fft(x, w, R, N)
  L  = length(x);
  M  = floor((L - N) / R) + 1;
  Xk = zeros(N, M);
  for m = 0 : M-1
    seg = x(m*R + (1:N)) .* w(:);                   % window a frame
    Xk(:, m+1) = fft(seg, N);                       % one N-pt FFT
  end
end

Two loops collapse to one + a library FFT. Same output matrix $X_k$ as Method A (up to floating-point noise). For the same 10 s example: $\approx 1.7\cdot 10^7$ ops — 100× faster.

三層迴圈降為兩層 + 一次 FFT。輸出 $X_k$ 與方法 A 完全相同（浮點誤差內）。同樣的 10 秒例子只需約 $1.7\cdot 10^7$ 次運算 — 快 100 倍。

Method C · Polyphase lite (shared exponentials)

方法 C · Polyphase 輕量版（共用指數）

% Reuse one twiddle matrix Wmat for every frame
function Xk = stft_polyphase_lite(x, w, R, N)
  L    = length(x);
  M    = floor((L - N) / R) + 1;
  Wmat = exp( -1j * 2*pi * (0:N-1).' * (0:N-1) / N );   % N×N DFT matrix
  Xk   = zeros(N, M);
  for m = 0 : M-1
    seg = x(m*R + (1:N)) .* w(:);
    Xk(:, m+1) = Wmat * seg;                            % matrix × vector
  end
end

Cost equals direct again — $O(N^2)$ per frame, not $O(N\log N)$ — but it illustrates the algebraic structure: STFT = "window × signal" multiplied by the DFT matrix. Polyphase proper (Ch 12) replaces the matrix with $O(N\log N)$ structured operators and is the route modern codecs take.

成本又變回 $O(N^2)$ 而非 $O(N\log N)$ — 但它把「STFT = (視窗×訊號) × DFT 矩陣」的代數結構直接寫出來。真正的 polyphase（Ch 12）會把此矩陣換成 $O(N\log N)$ 的結構化運算子，是現代 codec 採用的路線。

Sanity test

健全性測試

% Run all three methods, confirm they agree
fs = 44100; L  = fs * 2;  N = 512; R = 128;
x  = randn(L, 1);
w  = hann(N);

XA = stft_fb_direct(x, w, R, N);
XB = stft_fft(x, w, R, N);
XC = stft_polyphase_lite(x, w, R, N);

err_BA = max(abs(XB(:) - XA(:))) / max(abs(XA(:)));
err_CA = max(abs(XC(:) - XA(:))) / max(abs(XA(:)));
fprintf('||B - A||∞ = %.2e\n', err_BA);
fprintf('||C - A||∞ = %.2e\n', err_CA);
% Expected: both errors at machine epsilon ~ 1e-14.

If err_BA and err_CA aren't both at machine epsilon ($\sim 10^{-14}$), you've got a bug — likely a sign error on the exponential, a wrong hop indexing, or FFT-length mismatch with the window length. The agreement is the whole point: the FB view and the FFT view compute identical tensors.

若 err_BA、err_CA 沒有都在機器 epsilon（$\sim 10^{-14}$）等級，就是 bug — 常見是指數符號錯、跳距索引偏移、或 FFT 長度與視窗長度不一致。兩者一致正是重點：FB 詮釋與FFT 詮釋算出完全相同的張量。

Pick one mental model, measure the other. Think about modifications through the FB (per-band gain, per-band phase lock, cross-channel coupling) but compute with the FFT. That's the working-STFT-engineer's day-to-day workflow.

思考用一個模型、實測用另一個。設計修改時用 FB 詮釋（逐頻帶增益、逐頻帶相位鎖定、跨通道耦合）；計算時用 FFT。這就是實務 STFT 工程師的日常。
