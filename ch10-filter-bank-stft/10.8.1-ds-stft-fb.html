Stage 10-8-1 · Downsampled STFT Filter Bank (formal) 關卡 10-8-1 · 降取樣 STFT 濾波器組（正式定義） SASP §10.8.1 ↗

Write out the analysis and synthesis equations for the hop-$R$ STFT FB, nail the indexing conventions, and state the perfect-reconstruction condition in one box. This is the form you'll see in any modern codec spec (Vorbis, Opus, AAC).

把跳距 $R$ STFT FB 的分析與合成公式寫出來、固定索引慣例，並把完美重建條件濃縮成一個方框。所有現代編解碼規格（Vorbis、Opus、AAC）都採這個形式。

Analysis at hop $R$

跳距 $R$ 的分析

STFT (DOWNSAMPLED)
$$X_m[k]\;=\;\sum_{n=0}^{M-1} w[n]\,x[n + mR]\,e^{-j 2\pi k n/N}$$

$m$ is frame index, $R$ is hop (samples between frames), $M$ is window length, $N$ is DFT length (usually $\ge M$; if Portnoff, $N < M$). Output is a 2-D tensor $\{X_m[k]\}$ with time axis indexed by $m$ (rate $1/R$) and frequency axis indexed by $k$.

$m$ 為 frame 索引，$R$ 跳距（frame 間樣本數），$M$ 視窗長度，$N$ DFT 長度（通常 $\ge M$；若 Portnoff 則 $N < M$）。輸出為二維張量 $\{X_m[k]\}$，時間軸以 $m$（速率 $1/R$）、頻率軸以 $k$ 索引。

Synthesis (WOLA / FBS)

合成（WOLA / FBS）

WOLA INVERSE
$$\hat x[n]\;=\;\frac{1}{A}\sum_m w_s[n - mR]\,\cdot\,\underbrace{\frac{1}{N}\sum_{k=0}^{N-1} X_m[k]\,e^{j 2\pi k (n-mR)/N}}_{\text{IDFT of frame } m}$$

$w_s$ is the synthesis window (possibly equal to the analysis window $w$); $A$ is a normalisation constant. Read this as: for each frame, take the IDFT to get a time-domain snippet, multiply by $w_s$, overlap-add into the output. "WOLA" = weighted overlap-add.

$w_s$ 為合成視窗（可能等於分析視窗 $w$）；$A$ 為歸一常數。讀法：每 frame 取 IDFT 得時域小段、乘 $w_s$、疊加至輸出。"WOLA" = 加權重疊相加。

PR condition in one box

完美重建條件一方框

PR AT HOP R (ANY WINDOWS w, w_s)
$$\sum_m w_s[n - mR]\,w[n - mR]\;=\;A\quad\text{(const in }n\text{)}$$

This is the "COLA-on-product" condition. Special cases:

即「COLA 作用於乘積」條件。特例：

OLA (FBS-free): $w_s = 1$, so condition reduces to COLA-on-$w$.
WOLA with same window: $w_s = w$, condition becomes COLA-on-$w^2$ (square!). E.g. $\sqrt{\text{Hann}}$ on both sides is the classic "ola" in scipy.signal.stft.
Biorthogonal: pick $w_s$ as the dual of $w$ under COLA — always exists, always gives PR. Useful when $w$ is prescribed and you need to find the matching $w_s$. OLA（無 FBS）：$w_s = 1$，化為 $w$ 上的 COLA。
同視窗 WOLA：$w_s = w$，變成 $w^2$ 上的 COLA（平方！）。例如 scipy.signal.stft 預設兩側都是 $\sqrt{\text{Hann}}$。
雙正交：把 $w_s$ 選作 $w$ 在 COLA 下的對偶 — 永遠存在、永遠 PR。用於 $w$ 已給定、要找對應 $w_s$ 的場合。

Analysis / synthesis windows can be different. PR only needs their product to satisfy COLA at hop $R$. This 2-window freedom is what lets modern codecs (MDCT, Vorbis, Opus) use different windows on the two sides and trade analysis bandwidth for synthesis time-resolution.

分析與合成視窗可以不同。PR 只要兩者乘積在跳距 $R$ 下滿足 COLA 即可。現代編解碼（MDCT、Vorbis、Opus）靠這個兩視窗自由度換取「分析頻寬」與「合成時間解析度」。
