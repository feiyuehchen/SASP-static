Stage 10-8-2 · Downsampling with Anti-Aliasing 關卡 10-8-2 · 抗混疊的降取樣 SASP §10.8.2 ↗

Make the per-channel aliasing harmless by ensuring each channel's bandwidth is $\le 2\pi/R$. In the STFT FB this is a window shape requirement: the main-lobe width must not exceed $2\pi/R$. Practical windows for each hop are listed below.

讓每通道混疊無害的做法：保證每通道頻寬 $\le 2\pi/R$。在 STFT FB 中這是視窗形狀的要求：主波瓣寬不得超過 $2\pi/R$。以下列出各跳距對應的實用視窗。

The requirement

要求

ANTI-ALIAS CONDITION
$$|W(\omega)|\;\le\;\epsilon\quad\text{for }|\omega|\;\ge\;\pi/R$$

$\epsilon$ is your tolerance — the alias energy budget. For audible leak below $-60$ dB take $\epsilon = 10^{-3}$. Equivalently, side-lobe level must drop below $-60$ dB within the first $\pi/R$ of frequency. Combined with the need for some main lobe, this dictates the window length.

$\epsilon$ 是容忍度 — 混疊能量預算。若可聽閾值為 $-60$ dB，取 $\epsilon = 10^{-3}$。等價地，旁波瓣需在頻率 $\pi/R$ 內降至 $-60$ dB 以下。配合主波瓣需求，視窗長度就被決定。

Mapping hop $R$ to a reasonable window

跳距 $R$ 對應合理視窗的映射

$R = N$ (critically sampled): Running-sum rectangle. Sidelobe $-13$ dB inside $\pi/R=\pi/N$ — not enough for audible work. Good only for low-rate control-signal FBs.
$R = N/2$ (50% overlap): Hann or Hamming. Sidelobe $-32$ to $-43$ dB. The "default" for audio.
$R = N/3$: Blackman, $-58$ dB — pro-quality.
$R = N/4$: Blackman-Harris or Kaiser $\beta=8$, $-74$ to $-92$ dB — broadcast/forensic-grade.
$R \le N/8$: Extreme over-sampling, for pitch shifting that needs fine phase tracking. $R = N$（臨界取樣）：矩形累計和。旁波瓣 $-13$ dB（$\pi/R=\pi/N$ 範圍內），不夠應付可聽需求；只適合低速率控制訊號 FB。
$R = N/2$（50% 重疊）：Hann 或 Hamming。$-32$ 至 $-43$ dB。音訊「預設」。
$R = N/3$：Blackman，$-58$ dB — 專業品質。
$R = N/4$：Blackman-Harris 或 Kaiser $\beta=8$，$-74$ 至 $-92$ dB — 廣播/鑑識等級。
$R \le N/8$：極度過取樣，適合需要精細相位追蹤的變調。

Why not just filter harder?

為何不「濾更猛」就好？

A classical multirate decimator would put an anti-alias lowpass before the downsampler. In the STFT FB, the "anti-alias filter" is the analysis window itself — so making side-lobes lower means making the window longer (§10.7 Portnoff) or sharper-decaying (Blackman-Harris, Kaiser). There's no way to decouple the anti-alias job from the window; they are the same block.

古典多速率抽取器會在降取樣前放抗混疊低通。STFT FB 中「抗混疊濾波器」就是分析視窗 — 想讓旁波瓣更低就得把視窗拉長（§10.7 Portnoff）或衰減更快（Blackman-Harris、Kaiser）。無法把抗混疊與視窗分開；它們是同一塊。

Choose window to match hop. The hop sets the anti-alias margin; the window's side-lobes must cover that margin. Pairing a tight hop with a leaky window gives aliased artefacts you can actually hear.

選視窗要配合跳距。跳距決定抗混疊預算；視窗旁波瓣必須蓋住這個預算。小跳距配漏視窗會產生真能聽到的混疊失真。
