Zoom into the bank from §10.3.4 and nail down every index convention: input/output timing, the DC-centred vs. DC-left frequency axis, circular indexing $k \bmod N$, and the polyphase-friendly time axis. These look like bookkeeping — but mistakes here are why your first FB prototype sometimes sounds shifted or mirrored.

進一步細看 §10.3.4 的組，把索引慣例釘死：輸入/輸出時序、直流居中 vs. 直流在左的頻率軸、$k \bmod N$ 的循環索引、以及多相友善的時間軸。看起來只是記帳，但第一次實作 FB 原型常常「聽起來位移或鏡像」就是這裡出錯。

Index conventions we'll use

本站採用的索引慣例

Time $n$: integer, $-\infty < n < \infty$. Output frame index $m$, with $n = mR$ (hop $R$).
Bin $k$: $k \in \{0, 1, \ldots, N-1\}$. $\omega_k = 2\pi k/N$. For $k > N/2$ the frequency interpretation is $-(2\pi - \omega_k)$ — i.e. bin $k$ really means $\omega_k$ on $[0,2\pi)$ but reads as negative frequency on $(-\pi, \pi]$.
Windowed frame: $x_m[n] \triangleq x[n+mR]\,w[n]$ for $n\in\{0,\ldots,N-1\}$.
STFT entry: $X_m[k] = \sum_{n=0}^{N-1} x_m[n]\,e^{-j 2\pi k n / N}$. 時間 $n$：整數、$-\infty < n < \infty$。輸出 frame 索引 $m$、$n = mR$（跳距 $R$）。
Bin $k$：$k \in \{0, 1, \ldots, N-1\}$，$\omega_k = 2\pi k/N$。$k > N/2$ 時代表頻率為 $-(2\pi - \omega_k)$ — 即 bin $k$ 實為 $[0,2\pi)$ 上的 $\omega_k$，但對應 $(-\pi, \pi]$ 上的負頻。
加窗 frame：$x_m[n] \triangleq x[n+mR]\,w[n]$，$n\in\{0,\ldots,N-1\}$。
STFT 元素：$X_m[k] = \sum_{n=0}^{N-1} x_m[n]\,e^{-j 2\pi k n / N}$。

Frequency axis — DC-left vs DC-centred

頻率軸 — 直流在左 vs. 直流居中

Matlab/Octave fft outputs bins $k=0,\ldots,N-1$ with $k=0$ at DC and $k=N/2$ at Nyquist. This is "DC-left". For plotting, we typically fftshift to put DC in the middle so $k\in\{-N/2,\ldots,N/2-1\}$ — this is "DC-centred" and corresponds to $\omega\in[-\pi, \pi)$. The DFT FB machinery computes the first; the eye prefers the second.

Matlab/Octave 的 fft 輸出 $k=0..N-1$，$k=0$ 是直流、$k=N/2$ 是 Nyquist — 稱「直流在左」。畫圖時常 fftshift 把直流擺中間，使 $k\in\{-N/2,\ldots,N/2-1\}$，對應 $\omega\in[-\pi, \pi)$ — 稱「直流居中」。DFT FB 運算用前者；視覺呈現用後者。

The filter-bank form, written in matrix shape

矩陣形式的濾波器組

ONE FRAME AS MATRIX × VECTOR
$$\begin{bmatrix}X_m[0]\\X_m[1]\\\vdots\\X_m[N-1]\end{bmatrix}\;=\;\mathbf{W}_N\,\mathrm{diag}(w)\,\begin{bmatrix}x[mR]\\x[mR+1]\\\vdots\\x[mR+N-1]\end{bmatrix}$$

Here $\mathbf{W}_N$ is the standard $N\times N$ DFT matrix with entries $e^{-j 2\pi k n / N}$, and $\mathrm{diag}(w)$ is the diagonal matrix of window values. This compact form is what the FFT inside a plain stft() computes — exactly one matrix-vector product per frame (made cheap by the butterflies).

其中 $\mathbf{W}_N$ 是標準 $N\times N$ DFT 矩陣，元素 $e^{-j 2\pi k n / N}$；$\mathrm{diag}(w)$ 是視窗值組成的對角陣。這個緊湊形式就是 stft() 內部 FFT 在每 frame 做的事 — 一次矩陣-向量乘法（透過 butterfly 變便宜）。

Why bother drawing the FB twice?

為何要把 FB 畫兩次？

§10.2 showed the FB as "$N$ independent bandpass FIRs" (the pedagogical picture). §10.3.4 / §10.3.4.1 show it as "demodulate → LP → re-modulate" per channel (the algorithmic picture). They compute exactly the same $X_m[k]$. The two pictures are useful in different contexts: the first for thinking about what each bin captures, the second for writing efficient code and analysing polyphase / alias behaviour.

§10.2 把 FB 畫成「$N$ 個獨立帶通 FIR」（教學版）。§10.3.4 / §10.3.4.1 畫成「每通道解調變 → LP → 再調變」（演算法版）。兩者算出完全相同的 $X_m[k]$。兩張圖在不同場合各有用：第一張用於思考每個 bin 擷取什麼；第二張用於寫高效程式、分析多相與混疊行為。

Detail is destiny. Getting an STFT to round-trip cleanly at 1e-14 requires every index to agree: which sample is "0 of the frame", which bin is Nyquist, whether the DFT matrix uses $e^{-j...}$ or $e^{+j...}$. Spell every convention out once, code against it, and never mix conventions mid-function.

細節決定成敗。讓 STFT 在 1e-14 精度來回還原，必須所有索引一致：frame 的 0 是哪個樣本、哪個 bin 是 Nyquist、DFT 矩陣用 $e^{-j...}$ 還是 $e^{+j...}$。一次寫清慣例，照著寫程式，絕不在函式中途混用慣例。
