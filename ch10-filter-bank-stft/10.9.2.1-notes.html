Stage 10-9-2-1 · Points to Note on Time-Varying Modifications 關卡 10-9-2-1 · 時變修改的注意事項 SASP §10.9.2.1 ↗

Six practical reminders when shipping an STFT-based effect. Consider this the engineering checklist you run before calling an effect "done".

六條實務提醒 — 準備讓一個 STFT 效果「出貨」前該跑一遍的工程檢查表。

1. Use WOLA unless you have a reason not to

1. 沒特別理由就用 WOLA

Analysis-only windowing (OLA with $w_s = 1$) is cheaper but amplifies modification artefacts: modifications in one frame leak fully into the output with no synthesis-side smoothing. Symmetric WOLA ($w_s = w$ or $w = w_s = \sqrt{v}$) halves the artefact amplitude at minimal extra cost.

純分析加窗（OLA、$w_s = 1$）較便宜但會放大修改失真：某 frame 的修改會原封不動洩漏至輸出，合成端沒有平滑。對稱 WOLA（$w_s = w$ 或 $w = w_s = \sqrt{v}$）以極低額外成本把失真振幅對半砍。

2. Zero-pad each frame

2. 每 frame 零填充

Circular convolution artefacts at frame edges can be eliminated by zero-padding each frame by the length of the longest expected time-domain kernel of your modification. For typical EQ / compressor / pitch-shifter the kernel is $\le 10$ ms; zero-pad by that much (about 441 samples at 44.1 kHz).

每 frame 零填充「修改所對應的時域核心最長長度」可消除循環卷積邊界瑕疵。一般 EQ / 壓縮器 / 變調的核心 $\le 10$ ms；44.1 kHz 下補約 441 樣本。

3. Symmetric modifications for real output

3. 實值輸出需對稱修改

For real-valued output $y[n]$, the modification mask must obey the conjugate-symmetry rule $H_m[N-k] = H_m^*[k]$. Otherwise the ISTFT produces complex-valued $y$ (audible as DC offset after discarding the imaginary part, or as ultrasonic tones). Don't forget the Nyquist bin: $H_m[N/2]$ must be real.

實值輸出 $y[n]$ 需修改遮罩滿足共軛對稱 $H_m[N-k] = H_m^*[k]$。否則 ISTFT 會產生複值 $y$（丟虛部後聽起來像 DC 偏移或超音波）。別忘了 Nyquist bin：$H_m[N/2]$ 須為實數。

4. Normalise before, de-normalise after

4. 前歸一、後反歸一

FBS reconstruction requires dividing by $N\,w[0]$ (§10.1.3). Some libraries bake this into the istft; others expect you to do it. Test with an unmodified round-trip: $x[n] \to$ STFT $\to$ (no mod) $\to$ ISTFT $\to$ should give $x[n]$ within machine epsilon. If it gives $x[n]/N$ or $N\,x[n]$, fix the normalisation.

FBS 重建需除以 $N\,w[0]$（§10.1.3）。有些函式庫已內建於 istft；有些要你自己做。用未修改來回測試：$x[n] \to$ STFT $\to$（不修改）$\to$ ISTFT $\to$ 應在機器 epsilon 內等於 $x[n]$。若結果是 $x[n]/N$ 或 $N\,x[n]$，就是歸一化沒處理。

5. Phase needs its own attention

5. 相位要獨立處理

Magnitude-only modifications (reset phase to analysis phase) produce intelligible but "phasey" output. For critical work — pitch shifting, time-stretch, morphing — use a phase vocoder that tracks phase across frames (§11.5). The PR machinery in §§10.1–10.8 is blind to phase coherence; that's a separate problem.

純振幅修改（相位重設為分析相位）可理解但「phasey」。關鍵任務（變調、時間伸縮、morphing）需用跨 frame 追蹤相位的相位聲碼器（§11.5）。§10.1–10.8 的 PR 機制對相位一致性無感；這是另一層問題。

6. Validate on a sine sweep

6. 用掃頻正弦驗證

Before shipping: run a logarithmic sine sweep (20 Hz–20 kHz, 10 s) through the effect with modification off, check round-trip error is white / below $-100$ dB. Then turn modification on with a known target (e.g. $+6$ dB above 1 kHz) and verify the output spectrum matches. Catches most implementation bugs before users do.

出貨前：跑對數掃頻正弦（20 Hz – 20 kHz，10 秒），修改關閉時驗證來回誤差為白噪音且低於 $-100$ dB。再開啟修改並給已知目標（如 1 kHz 以上增益 $+6$ dB），檢查輸出頻譜是否吻合。多數實作 bug 會被這關抓到。

Ship with tests. Round-trip and sweep tests catch 90% of STFT mistakes. Automate them into CI; don't trust an ear alone.

測試隨發布。來回測試與掃頻測試能抓 90% 的 STFT bug。納入 CI，別只信自己的耳朵。
