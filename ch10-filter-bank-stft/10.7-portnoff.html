Stage 10-7 · Portnoff Windows 關卡 10-7 · Portnoff 視窗 SASP §10.7 ↗

Design windows longer than the DFT size $N$ that still satisfy Nyquist($N$). Credit to Portnoff (1976, 1981) — these are the windows that let the STFT FB use a long, low-side-lobe analysis window without growing the FFT size. Underpins modern SMS, phase vocoder, and speech coding work.

設計長度超過 DFT 大小 $N$、但仍滿足 Nyquist($N$) 的視窗。歸功於 Portnoff（1976, 1981）— 這類視窗讓 STFT FB 能用長、低旁波瓣的分析視窗而不必擴大 FFT。現代 SMS、相位聲碼器、語音編碼都以此為基礎。

Why go longer than $N$?

為何要比 $N$ 長？

Two pressures point the same way:

兩股壓力指向同一個方向：

1. Side-lobe level drops with window length. For a Kaiser with $\beta=10$, $-74$ dB needs roughly $M\ge 120$ samples — but typical FFT sizes $N$ for audio STFT are $512$ or $1024$. If we force $M=N$, we're under-sampled in the low-side-lobe regime.
2. FFT cost grows with $N\log N$. Doubling $N$ to accommodate a longer window doubles memory and nearly doubles CPU. Better to keep $N$ small and let $M > N$. 1. 旁波瓣隨視窗長度下降。Kaiser $\beta=10$ 達 $-74$ dB 大致需 $M\ge 120$ — 但音訊 STFT 常用 $N = 512$ 或 $1024$。若硬要 $M=N$，低旁波瓣區反而取樣不足。
2. FFT 成本隨 $N\log N$ 增長。把 $N$ 加倍以容納長視窗會讓記憶體加倍、CPU 近乎加倍。保持 $N$ 小、讓 $M > N$ 比較划算。

Portnoff's construction

Portnoff 的構造

Start with any smooth, low-side-lobe window $g[n]$ of length $M = L N$ (integer $L\ge 2$). Force the Nyquist($N$) condition by periodic summation:

從任意平滑、低旁波瓣的視窗 $g[n]$（長度 $M = L N$，整數 $L\ge 2$）出發。透過週期疊加強制 Nyquist($N$) 條件：

PORTNOFF RECIPE
$$w[n]\;=\;g[n]\,-\,\bigl(g[n] * \delta_{N}[n] - g[0]\delta[n]\bigr)$$

In practice the cleanest form is: write $g$ as a sum $g[n]=\sum_p a_p \phi[n - pN]$ where $\phi$ is any Nyquist($N$) template (e.g. the length-$N$ rect). Picking $a_p = g[pN]$ and enforcing $a_p=\delta[p]$ by subtracting a shifted impulse train gives a $w$ that vanishes at every non-zero multiple of $N$ — Nyquist($N$) by construction, with length $LN$ and the side-lobes inherited from $g$.

實務上最乾淨的形式：把 $g$ 寫成 $g[n]=\sum_p a_p \phi[n - pN]$，$\phi$ 為任意 Nyquist($N$) 模板（如長度 $N$ 矩形）。取 $a_p = g[pN]$ 並減去位移脈衝串讓 $a_p=\delta[p]$，就得到 $N$ 的每個非零倍數處為零、長度 $LN$ 的 $w$ — 設計出來即滿足 Nyquist($N$)，旁波瓣則承襲 $g$。

Analysis / synthesis with Portnoff windows

用 Portnoff 視窗做分析/合成

ANALYSIS
$$X_m[k]\;=\;\sum_{n} x[n]\,w[n - mR]\,e^{-j 2\pi k n/N}$$

Every term involving the window is evaluated modulo the $N$-point DFT — but since $w$ is length $LN$, each analysis frame folds $L$ time-aliased copies into the $N$-point FFT buffer:

視窗相關的每一項都在 $N$ 點 DFT 下計算 — 由於 $w$ 長度為 $LN$，每個分析 frame 會在 $N$ 點 FFT 緩衝區摺疊 $L$ 份時域混疊副本：

TIME-ALIASED FRAME (FOLD)
$$\tilde x_m[n]\;=\;\sum_{\ell=0}^{L-1} x[n + mR + \ell N]\,w[n + \ell N]\quad(n=0,\ldots,N-1)$$

Now do an $N$-point FFT on $\tilde x_m[n]$. The Nyquist($N$) property ensures the fold is PR-consistent: summing over channels $k$ (FBS) or over frame positions (OLA) recovers $x[n]$ exactly. The $L$-fold time-aliasing is not a loss — it's how the long window is encoded into the short FFT.

再對 $\tilde x_m[n]$ 做 $N$ 點 FFT。Nyquist($N$) 保證折疊與 PR 相容：對 $k$ 求和（FBS）或對 frame 位置求和（OLA）都能精確還原 $x[n]$。$L$ 倍時域混疊不是損失 — 而是「長視窗編碼進短 FFT」的方式。

Practical trade-off

實務取捨

+ Side-lobes decouple from FFT size — use a 1024-pt FFT with a length-4096 Kaiser at $\beta=10$ for $-74$ dB leakage.
+ Frequency resolution improves: main-lobe width scales as $1/M$, not $1/N$.
− Extra memory to hold the longer window and its past $L$ frames.
− Analysis buffering introduces $M/2$ extra latency, not $N/2$. Real-time systems must budget for this. + 旁波瓣與 FFT 大小解耦 — 1024 點 FFT 可配長度 4096 的 $\beta=10$ Kaiser，洩漏壓到 $-74$ dB。
+ 頻率解析度提升：主波瓣寬度 $1/M$、不是 $1/N$。
− 要多存長視窗與過去 $L$ 份 frame。
− 分析緩衝造成 $M/2$ 延遲（非 $N/2$）。即時系統需預留此預算。

Portnoff = long window, short FFT, still PR. The Nyquist($N$) condition is what makes time-aliasing benign: the fold that would normally destroy the signal instead encodes it losslessly. This is the mathematical trick underneath every modern phase-vocoder / SMS analysis stage.

Portnoff = 長視窗 + 短 FFT，仍 PR。Nyquist($N$) 讓時域混疊變無害：原本會毀掉訊號的折疊，改為無損編碼。這就是現代相位聲碼器 / SMS 分析階段背後的數學技巧。
