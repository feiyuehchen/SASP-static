Stage 10-9 · STFT with Modifications 關卡 10-9 · 加上修改的 STFT SASP §10.9 ↗

All the PR machinery up to §10.8 assumed we pass $X_m[k]$ straight through to synthesis. The point of the STFT, in audio, is to modify $X_m[k]$ before resynthesis — that is where equalisers, pitch shifters, harmonisers, spectral denoisers, spectral freeze, and vocoders live. This stage frames what "modification" does to PR, and splits the options into fixed (§10.9.1) and time-varying (§10.9.2).

§10.8 之前的 PR 機制都假設 $X_m[k]$ 原封不動送去合成。STFT 在音訊上的價值恰恰是在重合成前修改 $X_m[k]$ — 等化器、變調、harmoniser、頻譜降噪、頻譜凍結、聲碼器都在這層運作。本關界定「修改」對 PR 的影響，並把選項分為固定（§10.9.1）與時變（§10.9.2）兩類。

The modified pipeline

加上修改的流水線

MODIFIED STFT 
x[n] → [analysis] → X_m[k] → [MODIFY: H_m[k]] → Y_m[k] → [synthesis] → y[n]
                                   ↑
                         per-channel / per-frame gain

The modification layer multiplies each STFT tensor element by a (possibly frame- and bin-dependent) factor $H_m[k]$:

修改層把 STFT 張量每個元素乘上（可能隨 frame 與 bin 變化的）因子 $H_m[k]$：

$$Y_m[k]\;=\;H_m[k]\,\cdot\,X_m[k]$$

If $H_m[k] = H[k]$ (no dependence on $m$), the modification is time-invariant; the output $y[n]$ is simply $x[n]$ filtered by an LTI system whose DTFT matches $H[\cdot]$ on the DFT grid (§10.9.1). If $H_m[k]$ changes with $m$, you get time-varying filtering — powerful, but with subtle pitfalls covered in §10.9.2.

若 $H_m[k] = H[k]$（不依 $m$），即時不變修改；輸出 $y[n]$ 等於 $x[n]$ 通過某 LTI 系統（其 DTFT 在 DFT 網格上等於 $H[\cdot]$）— §10.9.1。若 $H_m[k]$ 隨 $m$ 變，就是時變濾波 — 威力強，但坑多，§10.9.2 會詳述。

Three examples

三個例子

(1) Fixed EQ: $H[k] = G_k$ — apply a per-bin gain. Same as LTI filtering; works cleanly at any PR hop.
(2) Noise gate: $H_m[k] = 1$ if $|X_m[k]| > \text{threshold}$, $0$ otherwise. Frame-dependent, magnitude-only. Watch for zipper artefacts from sudden switching.
(3) Pitch shift: scale frequencies per frame; needs phase vocoder phase tracking. Classic time-varying modification with subtle alias risk. (1) 固定 EQ：$H[k] = G_k$ — 每 bin 給一個固定增益。等同 LTI 濾波；任何 PR 跳距下都乾淨。
(2) 噪音閘：$H_m[k] = 1$ 當 $|X_m[k]| >$ 門檻、否則 $0$。隨 frame 變、只看振幅。注意突變會產生 zipper 雜音。
(3) 變調：每 frame 縮放頻率；需相位聲碼器追蹤相位。經典的時變修改，容易產生混疊。

PR protects against analysis/synthesis mismatch, not against modification mistakes. Even with a perfect hop/window pair, a bad modification (mid-frame jumps, unpaired bin gains, non-causal time-varying effects) will produce artefacts. The rest of §10.9 shows how to stay inside the safe region of modifications.

PR 只保護「分析/合成不匹配」，不保護「修改做錯」。即使視窗與跳距完美，修改若做糟（frame 中跳變、bin 不成對增益、非因果時變效果）照樣產生失真。§10.9 其他節說明如何停留在「安全的修改區」內。
