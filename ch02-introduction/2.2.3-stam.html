Stage 2-2-3 · Short-Time Analysis, Modification, Resynthesis 關卡 2-2-3 · 短時分析、修改、再合成 SASP §2.2.3 ↗

The STFT is a round trip. Go to the frequency domain, change something on the way, come back. The changes can be time-varying.

STFT 是一趟來回路程。去頻域一趟，路上改一點東西，再回來。改的東西可以隨時間變。

The AMR pipeline

「分析-修改-再合成」管線

The bookkeeping looks like:

流程的記帳法長這樣：

AMR PIPELINE
$$x[n]\xrightarrow[\text{window+FFT}]{\text{analysis}}X_m(\omega)\xrightarrow[\text{multiply,filter,...}]{\text{modify}}\tilde X_m(\omega)\xrightarrow[\text{IFFT+OLA}]{\text{resynth}}\tilde x[n]$$

Choose a analysis window and a hop so the overlap-add path is lossless (the COLA condition). Then each frame's spectrum can be shaped — filtered, pitch-shifted, noise-reduced, cross-synthesised — before it is inverse-FFT'd and stitched back together.

先選一組分析視窗與跳距， 讓重疊相加路徑是無損的（ COLA 條件）。 然後就能在每個 frame 的頻譜上動手腳 — 濾波、變調、去噪、交叉合成 — 再反 FFT、拼接回時域。

Beware the phasing artifact: if you modify each frame's phase independently, consecutive frames can disagree, giving a shimmery / metallic sound. Good systems enforce phase continuity across frames.

小心相位跳動： 如果每個 frame 的相位獨立處理，前後 frame 對不上，會聽到金屬質感的晃動。好的系統會跨 frame 約束相位連續。
