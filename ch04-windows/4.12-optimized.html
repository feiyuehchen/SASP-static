Stage 4-12 · Optimized Windows 關卡 4-12 · 最佳化視窗 SASP §4.12 ↗

Instead of starting from a named formula (Hann, Kaiser, Dolph-Chebyshev, …), directly state the desired spectrum properties — main-lobe width, side-lobe floor, smoothness, OLA constraints — as constraints in a numerical optimisation. This gives you windows tailored to specific applications (audio codec, filter-bank, STFT with prescribed aliasing) rather than a catalog-lookup.

不從已命名公式（Hann、Kaiser、Dolph-Chebyshev …）出發 — 把主瓣寬、旁瓣地板、平滑度、OLA 約束直接陳述為數值最佳化的約束。這樣可針對特定應用（音訊編碼、濾波器組、有限混疊 STFT）量身定做視窗，而非查表取用。

TYPICAL OPTIMISATION SHAPE
minimize: $\max_{\omega\in\text{stop}}|W(\omega)|$ (side-lobe peak)
subject to: $|W(\omega)|$ main-lobe shape, $w[n]$ symmetry,
    OLA property $\sum_k w[n-kR]=C$, and/or smoothness bounds. 最小化：$\max_{\omega\in\text{stop}}|W(\omega)|$（旁瓣峰值）
約束：$|W(\omega)|$ 主瓣形狀、$w[n]$ 對稱、
    OLA 性質 $\sum_k w[n-kR]=C$、以及／或平滑度界。

Every modern audio codec uses a window designed this way. MP3, AAC, Vorbis, Opus, SBC — all derived via constrained optimisation, not pulled from a textbook table. §4.12.1 shows which constraints each codec chose; §4.12.2 gives the general recipe.

所有現代音訊編碼器都用這條路設計視窗。MP3、AAC、Vorbis、Opus、SBC — 全是約束最佳化的產物，而非教科書表格。§4.12.1 說明各編碼器選了哪些約束；§4.12.2 給出通用食譜。
