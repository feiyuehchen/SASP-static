Built in as hamming(N). Or a one-liner:

內建函式 hamming(N)；一行版：

% Matlab / Octave
w = 0.54 - 0.46 * cos(2*pi*(0:N-1)'/(N-1));

% Python / NumPy
w = 0.54 - 0.46 * np.cos(2*np.pi*np.arange(N) / (N-1))

% JavaScript (this site)
const w = DSP.WINDOWS.hamming(N);

Values 0.54 and 0.46 are rounded — the mathematically optimal Hamming uses 0.53836 and 0.46164. The 0.54/0.46 pair is the traditional round-number version; the difference is tiny.

0.54 與 0.46 其實是近似；數學最優的 Hamming 是 0.53836 / 0.46164。0.54 / 0.46 是傳統好記版本，差異極小。
