Stage 4-12-1 · Windows for Audio Coding 關卡 4-12-1 · 音訊編碼用視窗 SASP §4.12.1 ↗

A good codec window must satisfy all four simultaneously:

好的編碼器視窗必須四條同時滿足：

  • Perfect reconstruction after 50 % overlap-add — squared-sum must be constant: $\sum_k w^2[n-kR] = \text{const}$ (so-called Princen-Bradley condition).
  • 完美還原（50% 重疊相加後）— 平方和須為定值 $\sum_k w^2[n-kR] = \text{const}$（Princen-Bradley 條件）。
  • Smooth main lobe matching psychoacoustic masking — sharp cutoffs create audible pre-echo.
  • 主瓣平滑、符合聽覺遮蔽 — 太陡會產生可聞預迴響。
  • Low side-lobes to stop quantisation noise in one band leaking into another.
  • 旁瓣夠低，避免量化雜訊跨帶洩漏。
  • Short-enough latency — block size usually 1024 to 2048 samples.
  • 延遲可接受 — 方塊大小通常 1024 到 2048 樣本。

Codec roll call

編碼器點名

MP3 / AAC: MLT sine $w[n]=\sin\!\bigl(\tfrac{\pi(n+1/2)}{N}\bigr)$ — satisfies $\sin^2+\cos^2=1$ COLA.
Vorbis: Windowed MLT sine, $w[n]=\sin\!\bigl(\tfrac{\pi}{2}\sin^2\!(\tfrac{\pi(n+1/2)}{N})\bigr)$ — deeper sides, still COLA.
Opus: switches between Vorbis window (stationary) and short transient-tuned window.
Bluetooth SBC: square-root Hann + cosine tweaks — matches the SBC filter bank's analysis-synthesis constraints. MP3 / AAC：MLT 正弦 $w[n]=\sin\!\bigl(\tfrac{\pi(n+1/2)}{N}\bigr)$ — 滿足 $\sin^2+\cos^2=1$ COLA。
Vorbis：套用 MLT 正弦，$w[n]=\sin\!\bigl(\tfrac{\pi}{2}\sin^2\!(\tfrac{\pi(n+1/2)}{N})\bigr)$ — 旁瓣更深，仍 COLA。
Opus：穩態用 Vorbis 視窗，瞬態切到短視窗。
Bluetooth SBC：根 Hann + cos 調整 — 配合 SBC 濾波器組的解析/合成約束。

Codec windows are solutions to a system. Four constraints — PR + smoothness + side-lobes + latency — collapse the design space to a narrow family. Named codec windows are named solutions to that system, not separate inventions.

編碼器視窗是系統解。四條約束 — PR + 平滑 + 旁瓣 + 延遲 — 把設計空間壓到一小家族。各編碼器的視窗都是這個系統的解、非個別發明。

Q. Why must audio codec windows satisfy the squared-OLA (Princen-Bradley) condition, not just linear OLA?

問：為何音訊編碼器視窗需滿足平方 OLA（Princen-Bradley），而非僅線性 OLA？

Lab · MLT sine — the squared-OLA check

實驗 · MLT 正弦 — 平方 OLA 驗證

Three copies of $\sin^2(\pi(n+1/2)/N)$ overlapped at 50% hop. Blue + yellow + pink individual shapes; green trace: their sum — exactly constant = 1 in the middle. That flat green is why MP3/AAC/Vorbis work.

三個 $\sin^2(\pi(n+1/2)/N)$ 複本在 50% 跳幀重疊。藍+黃+粉為個別形狀；綠線為總和 — 中段恰為定值 1。綠線平坦就是 MP3/AAC/Vorbis 能用的原因。
