Matlab ships both bartlett(N) and triang(N). Same shape, slightly different endpoints.

Matlab 有 bartlett(N) 與 triang(N) 兩個內建。形狀相同、端點略有不同。

% Matlab / Octave
w = bartlett(N);   % endpoints = 0
w = triang(N);     % endpoints = 1/N

% Manual
n = (0:N-1)';
M = (N-1)/2;
w = 1 - abs((n - M) / M);

% Python / NumPy
n = np.arange(N)
M = (N-1)/2
w = 1 - np.abs((n - M) / M)

% JavaScript (this site)
const w = DSP.WINDOWS.bartlett(N);

bartlett vs. triang

bartlett vs. triang

bartlett(N): $w[0] = w[N-1] = 0$. The strict "triangle between zeros" interpretation.
triang(N): $w[0] = w[N-1] = 1/N$. Treats the window as $N$ samples of a slightly wider continuous triangle, avoiding the lossy endpoint zeros.

For general spectrum analysis the difference is negligible. For precise coherent-gain or periodogram normalisation, pick one and stick with it. bartlett(N)：$w[0] = w[N-1] = 0$。嚴格「兩端為零的三角形」定義。
triang(N)：$w[0] = w[N-1] = 1/N$。視為「稍寬連續三角」的 $N$ 點取樣，避免端點為零造成的能量損失。

一般頻譜分析差異可忽略。要計算精確相干增益或 periodogram 歸一化時，選一個用到底即可。

Two variants, one concept. If two function names give almost-identical windows, pick the one your downstream code expects.

兩個名字、一個概念。若兩個函式給出幾乎相同的視窗，選下游程式碼期望的那個。
